Core Principles

Prinsip yang bikin Cursor Agent terasa waras

Kalau enam prinsip ini dipegang, hasil Cursor biasanya lebih stabil: lebih sedikit rework, lebih sedikit konteks mubazir, dan lebih jelas kapan harus naik ke fitur agent yang lebih kuat.

Start with context, not code

Mode `Ask` paling aman untuk orientasi awal: pahami boundary, cari file, dan minta peta area kerja dulu sebelum membiarkan agent mengubah banyak file.

Docs mode Cursor menggambarkan Ask sebagai read-only exploration.
Choose the mode on purpose

`Agent` untuk pekerjaan otonom lintas file, `Manual` untuk edit presisi, `Ask` untuk diagnosis, dan `Custom Modes` untuk setup berulang yang sudah terbukti cocok.

Jangan semua hal dilempar ke Agent hanya karena bisa.
Auto is a legitimate default

Saat belum yakin mau model apa, `Auto-select` adalah pilihan aman. Docs Cursor menyebutnya sebagai default yang solid jika Boss tidak yakin harus memilih apa.

Bagus untuk menjaga flow tanpa overthinking model di tiap chat.
Keep context intentionally small

Chat yang kegendutan bikin agent makin mahal dan makin liar. Reset thread saat fase kerja berubah, dan taruh instruksi stabil di rules atau `AGENTS.md`.

Rules memang dirancang sebagai persistent prompt-level context.
Escalate features, do not spam them

`/multitask`, `worktrees`, `multi-root`, `/loop`, dan automations sangat kuat, tapi paling sehat dipakai ketika pekerjaan memang paralel, terisolasi, atau berulang.

Feature baru 2026 itu multiplier, bukan default wajib.
Verification closes the loop

Agent yang terlihat pintar tetap harus diverifikasi. Build, test, focused lint, dan runtime repro kecil tetap jadi penutup yang sehat sebelum close-out.

Auto-review Run Mode mengurangi friction, bukan menggantikan verifikasi.
Daily Workflow

Workflow Cursor Agent yang paling sehat untuk kerja harian

Urutan ini paling masuk akal untuk repo nyata: pahami dulu, edit seperlunya, baru eskalasi ke fitur yang lebih otonom atau lebih mahal saat ada alasan yang jelas.

Warm Up

Bersihkan konteks dan boundary dulu

Pastikan repo terindeks, rules dan `AGENTS.md` relevan, lalu minta agent menjelaskan area kerja, file penting, dan risiko sebelum menyentuh implementasi.

Ask First

Mulai di Ask untuk orientasi

Gunakan `Ask` untuk mapping codebase, minta shortlist file, atau ringkasan dependency path. Ini mengurangi edit ngawur saat nanti pindah ke Agent.

Agent Next

Pindah ke Agent saat scope sudah cukup jelas

Begitu batas tugas mulai rapi, barulah pakai `Agent` untuk implementasi, refactor terarah, menjalankan command, dan memperbaiki error dengan loop kerja yang lebih otonom.

Parallelism

Pakai `/multitask` atau worktrees hanya untuk kerja paralel nyata

Fitur tanggal 24 April 2026 ini ideal saat ada sub-tugas independen, branch terisolasi, atau perubahan lintas repo yang memang tidak enak dikerjakan serial.

Long Runs

Pakai `/loop`, automations, atau cloud agent untuk kerja berulang

Kalau task-nya polling status, cek deploy, atau automasi berbasis event, jangan paksa chat biasa. Pakai loop lokal, automation, atau surface seperti Jira jika itu sumber kerja aslinya.

Close

Tutup dengan verifikasi dan handoff singkat

Minta agent merangkum apa yang berubah, lalu jalankan test, build, atau repro kecil yang paling relevan. Jangan tutup thread dengan narasi tanpa evidence.

Tips & Tricks

Kebiasaan power-user yang bikin Cursor lebih efisien

Ini bagian yang paling terasa efeknya di hari ke hari: lebih sedikit prompt berulang, lebih sedikit approval yang melelahkan, dan lebih tajam saat memilih surface kerja.

Rules + AGENTS.md Taruh instruksi stabil di tempat yang persisten
  • Project rules untuk kebijakan repo, style, dan boundary.
  • `AGENTS.md` untuk operating contract yang mudah dibaca agent.
  • Jangan copy-paste instruksi panjang di setiap thread kalau bisa dibuat reusable.
Auto-select + Custom Modes Jadikan kombinasi yang berhasil sebagai preset
  • `Auto-select` aman saat Boss belum yakin model mana yang cocok.
  • Kalau sudah ketemu pola yang pas, simpan sebagai `Custom Mode` dengan model dan instruction yang sesuai.
  • Ini lebih hemat daripada mengulang prompt yang sama terus-menerus.
Run Mode Paksa friction hanya di tempat yang memang berisiko
  • Update `3.6` pada 29 Mei 2026 menambahkan `Auto-review Run Mode` untuk shell, MCP, dan fetch.
  • Bagus untuk kerja lebih lama dengan approval lebih sedikit, tapi tetap butuh allowlist dan boundary yang waras.
  • Anggap ini akselerator, bukan alasan untuk membiarkan agent bebas total.
Surface Match Pilih permukaan kerja yang sesuai sumber masalah
  • Gunakan editor dan Agents Window untuk coding langsung.
  • Gunakan Jira integration untuk tiket yang memang lahir dari Jira.
  • Gunakan automations untuk pemantauan atau tugas event-driven, bukan chat manual berulang.
Artifacts Pakai canvas saat output perlu dibaca orang lain
  • Canvases cocok untuk report, dashboard, atau interface hasil kerja agent.
  • Shared canvases dari 20 Mei 2026 enak untuk handoff tanpa membagikan seluruh thread.
  • Pilih ini saat output-nya lebih penting daripada percakapan mentahnya.
Context Hygiene Potong thread lama sebelum ia memotong kualitas
  • Satu thread untuk satu fase kerja lebih sehat daripada satu thread untuk segalanya.
  • Thread baru lebih baik saat tujuan berubah dari audit ke implementasi, atau dari implementasi ke release verification.
  • Ini sering lebih murah daripada memaksa Max Mode di thread lama.
Anti-Patterns

Hal yang paling sering bikin Cursor terasa mengecewakan

Biasanya bukan modelnya yang jelek, tapi workflow-nya yang salah. Empat pola ini paling sering bikin hasil berantakan, mahal, atau misleading.

Ambiguous start Langsung suruh Agent implement tanpa orientasi
  • Akibatnya agent mencari jalur tercepat versinya sendiri.
  • Untuk task abu-abu, selalu mulai dari pertanyaan orientasi lebih dulu.
Mode mismatch Menggunakan Agent untuk edit kecil yang butuh presisi
  • `Manual` atau prompt yang lebih terarah sering lebih bagus untuk edit kecil.
  • Otonomi tinggi tidak selalu sama dengan hasil yang lebih presisi.
Feature overkill Memakai Max, `/multitask`, atau `/loop` untuk masalah yang tidak membutuhkannya
  • Parallelism, long-running loops, dan context jumbo itu powerful, tapi ada cost dan complexity-nya.
  • Pakai hanya jika task memang paralel, berulang, atau sangat context-heavy.
No evidence close-out Menutup task berdasarkan narasi, bukan bukti
  • Ringkasan agent yang rapi tetap bukan pengganti `test`, `build`, atau repro langsung.
  • Ini kesalahan paling mahal karena kelihatan selesai padahal belum aman.
Prompt Patterns

Snippets yang enak dipakai langsung di Cursor

Bukan template sakti, tapi pola prompt yang cukup pendek dan biasanya memberi agent jalur kerja yang lebih sehat.

Ask Mode Prompt orientasi awal
  • Jelaskan boundary area ini, file kunci yang perlu saya lihat, dan risiko kalau saya edit sekarang. Jangan ubah file dulu.
  • Bagus untuk repo yang belum familiar atau task yang masih kabur.
Agent Mode Prompt implementasi terarah
  • Implement perubahan ini dengan scope sekecil mungkin, jelaskan rencana singkat, lalu jalankan verifikasi paling relevan sebelum merangkum hasilnya.
  • Bagus untuk fix atau refactor yang sudah jelas boundary-nya.
/multitask Prompt saat tugas bisa dipecah
  • Pecah pekerjaan ini menjadi sub-tugas independen dan kerjakan paralel hanya jika tidak saling konflik. Satukan hasil dan sisa risiko di akhir.
  • Pakai hanya jika benar-benar ada paralelisme nyata.
/loop Prompt untuk tugas yang berulang
  • Pantau status deploy ini setiap 5 menit sampai sehat, lalu hentikan loop dan ringkas hasilnya.
  • Contoh resmi changelog memang mengarah ke pola seperti ini.
Model escalation Prompt saat mau naik model
  • Mulai dari Auto. Kalau konteks terlalu besar atau reasoning terlalu dangkal, rekomendasikan model berikutnya dan alasan teknisnya sebelum lanjut.
  • Membantu mengontrol cost dan mengurangi impuls langsung ke model termahal.
Close-out Prompt penutup yang sehat
  • Ringkas apa yang berubah, apa yang diverifikasi, apa yang belum diverifikasi, dan risiko sisa tanpa melebih-lebihkan status selesai.
  • Bagus untuk menjaga honesty pada akhir thread.
Model Guide

Roster model Cursor yang current per 30 Mei 2026

Section ini sekarang memakai route resmi `models-and-pricing` yang current, bukan daftar lama yang dulu masih berpusat di `GPT-4.1`, `o3`, `Gemini 2.5 Pro`, atau `Claude 4 Sonnet` saja. Tujuannya tetap ringkas: roster yang aktual plus pembacaan operasional yang cukup untuk keputusan cepat.

Safe default

`Auto-select` tetap paling aman kalau Boss belum punya preferensi model atau hanya ingin tetap mengalir.

Bagus saat roster model berubah cepat dan Boss tidak ingin mengurusi routing manual setiap saat.
OpenAI coding lane

`GPT-5.3 Codex` sekarang jauh lebih relevan daripada referensi lama. Meta docs menyebutnya sebagai top benchmark scorer dan cost-effective all-rounder.

Ini kandidat paling masuk akal untuk coding all-rounder yang current.
Highest persistence

`GPT-5.5` kini muncul sebagai model OpenAI paling cerdas di roster current, dengan deskripsi resmi stronger persistence.

Masuk akal saat tugas panjang butuh konsistensi lebih besar.
Balanced Claude lane

`Claude 4.6 Sonnet` sekarang lebih relevan daripada referensi Sonnet lama. Docs menyebutnya balanced Claude model dengan thinking support.

Dari page resmi: 200k default context, bisa diperluas ke 1M di Max Mode.
Fast / compact lane

Roster current menonjolkan `GPT-5 Fast`, `GPT-5 Mini`, `GPT-5.4 Mini`, `GPT-5.4 Nano`, dan `Claude 4.5 Haiku` sebagai jalur ringan atau cepat.

Cocok untuk triage, drafting, dan task yang tidak layak membakar model berat.
New provider spread

Cursor current roster sekarang juga mencakup `Gemini 3.x`, `Grok 4.x`, `Grok Build 0.1`, `Kimi K2.5`, dan keluarga `Cursor Composer`.

Jadi section model handbook harus dibaca sebagai moving roster, bukan daftar permanen.
