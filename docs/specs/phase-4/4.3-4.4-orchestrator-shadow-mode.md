# Phase 4: Sub-Task 4.3 & 4.4 — Orchestrator Gateway API & Shadow Mode

## 4.3: Orchestrator Gateway API (NestJS)

### Architecture Rule
ALL applications (Referralink, Simulator, Abyss Dashboard) MUST execute flows via:
`POST /flows/:flowId/run` → Orchestrator → Langflow → aiSession log → return

### Bootstrap
```bash
pnpm create-app orchestrator --template nestjs
cd apps/orchestrator
pnpm add @nestjs/common @nestjs/core @nestjs/websockets socket.io \
  @the-abyss/langflow-client @the-abyss/database \
  @the-abyss/shared-types class-validator class-transformer \
  @opentelemetry/api @opentelemetry/sdk-node
```

### Core Components (Drafts)

#### Flows Controller — `src/flows/flows.controller.ts`
(See original MDC for full implementation details)

#### Flows Service — `src/flows/flows.service.ts`
*   **Mandatory:** Cost limit check before execution.
*   **Mandatory:** Log every execution to `aiSession` (Abyss Dashboard).
*   **Mandatory:** Compliance audit log for domain-specific requirements.

## 4.4: Shadow Mode A/B Testing Framework

### Shadow Mode Service — `src/flows/shadow-mode.service.ts`
*   Executes primary and shadow flows in parallel.
*   Always returns primary result.
*   Logs comparison (match, latency, cost) for monitoring.

### Rollback Service — `src/flows/rollback.service.ts`
*   Monitors agreement rate (threshold: 90%).
*   Auto-triggers rollback if threshold is breached.

---

## Success Criteria
- [ ] `POST /flows/:flowId/run` operational.
- [ ] All executions logged to `aiSession`.
- [ ] Latency p99 < 500ms (excluding LLM).
- [ ] Compliance audit log for active flows.
- [ ] Rollback auto-triggers when agreement < 90%.
