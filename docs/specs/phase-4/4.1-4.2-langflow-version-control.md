# Phase 4: Sub-Task 4.1 & 4.2 — Langflow Version Control & Custom Components

## 4.1: Langflow Version Control Setup

### Objective
Establish Git as the single source of truth for all Langflow flow definitions. 
Every flow in Langflow MUST have a corresponding versioned JSON in Git.

### Flow Metadata Schema
`flows/definitions/{domain}/metadata.yaml` must follow the schema:
- `flowId`, `domain`, `version`, `owner`, `status` (production/staging/rollback), `testCoverage`, `costPerRun`.

### Sync Flows CLI
`flows/scripts/sync-flows.ts` — Automates exporting flows from Langflow to Git.

### Git Hooks
- `pre-commit`: Validate JSON schema.
- `post-merge`: Auto-sync to staging.

## 4.2: Custom Domain-Specific Langflow Components

### Python Components
- **FHIRValidatorComponent**: Validates FHIR resources (Patient, Observation) against profiles.
- **MedicalCodeLookupComponent**: Resolves SNOMED CT, LOINC, RxNorm codes.
- **SentimentAnalyzerComponent**: Classifies academic feedback.

### JavaScript Components
- **FHIRBundleBuilderComponent**: Constructs FHIR Bundles from individual resources.
