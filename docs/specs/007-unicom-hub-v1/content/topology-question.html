A — Hub & Spoke

Semua agent bicara melalui UNICOM Hub terpusat. Hub menjadi router, registry, dan message broker sekaligus.

✅ Mudah dimonitor    ✅ Security terpusat
⚠️ Single point of failure    ⚠️ Latency +1 hop

tiap agent kenal semua agent lain

B — Full Mesh (P2P)

Setiap agent bisa langsung kontak agent lain tanpa perantara. Agent punya address book berisi semua agent yang terdaftar.

✅ Benar-benar "DIRECT"    ✅ Latensi minimum
⚠️ Kompleks scaling    ⚠️ Sulit di-debug
📋 Registry (lookup only)
"di mana Agent B?" → dapat address
Agent A ⟷ Agent B
↑ komunikasi langsung setelah lookup

C — Registry + Direct Call (Recommended)

Registry ringan hanya untuk "siapa ada di mana", tapi pesan dikirim langsung agent-ke-agent tanpa proxy.

✅ Direct tapi terorganisir    ✅ Scalable
✅ Mudah debug    ⚠️ Perlu maintain registry
