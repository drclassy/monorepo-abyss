Section 2 — Message Protocol & Agent Status

Struktur data yang mengalir melalui UNICOM Hub

Agent Status Lifecycle

disconnected → ● connected → ◎ idle ⇄ ⟳ streaming ⇄ ⚙ busy
Status otomatis di-broadcast ke semua agents ketika berubah
// AgentEntry — disimpan di in-memory registry Hub
interface AgentEntry {
  id: string                  // "claude-code" | "roocode" | "codex"
  displayName: string         // "Claude Code 1.x"
  capabilities: string[]       // ["code-review", "file-edit", "search"]
  status: "connected" | "idle" | "streaming" | "busy"
  connectedAt: number          // unix timestamp
  lastSeen: number             // heartbeat timestamp
}

Pesan yang Mengalir Antar-Agent

// UNICOMMessage — setiap pesan yang diroute Hub
interface UNICOMMessage {
  id: string               // UUID v4
  from: string              // sender agent id
  to: string | "broadcast"   // target agent or all
  content: string           // message body (markdown ok)
  type: "message" | "status_update" | "ack"
  replyTo?: string           // thread reference (optional)
  timestamp: number
}

MCP Tools yang Diekspos (5 tools, +1 dari sebelumnya)

register_agent(id, displayName, capabilities[]) → AgentEntry update_status(id, status: "idle"|"streaming"|"busy") → broadcast status_update send_message(to, content, replyTo?) → UNICOMMessage (SSE push to target) broadcast(content) → UNICOMMessage (SSE push to all) list_agents() → AgentEntry[] with live status
