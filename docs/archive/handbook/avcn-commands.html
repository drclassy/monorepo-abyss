Slash Commands
Command Skill Kapan Pakai
/avcn-brainstorm PLANbrainstorming Sebelum nulis kode apapun — eksplorasi, ajukan 2–3 opsi ke Chief. Hard gate: dilarang langsung kode.
/avcn-write-plan PLANwriting-plans Brainstorm disetujui — buat HANDOFF.md step-by-step dengan rollback plan.
/avcn-execute-plan PLANexecuting-plans Setelah Chief ketik GO — jalankan plan per task, review tiap batch.
/avcn-tdd CODEtest-driven-development Tulis failing test dulu, baru implementasi. Iron law: merah dulu, baru hijau.
/avcn-subagent CODEsubagent-driven-development Task besar — dispatch subagent per step, review setiap task selesai.
/avcn-parallel CODEdispatching-parallel-agents Ada 3+ bug/failure independent sekaligus — selesaikan semua bersamaan.
/avcn-worktree GITusing-git-worktrees Sebelum mulai feature baru — isolated workspace, cegah branch saling ganggu.
/avcn-finish GITfinishing-a-development-branch Tests hijau, done — pilih: merge / PR / simpan / discard. Worktree cleanup otomatis.
/avcn-request-review REVIEWrequesting-code-review Setelah tiap task di subagent workflow — dispatch code-reviewer, catch issues lebih awal.
/avcn-receive-review REVIEWreceiving-code-review Dapat feedback reviewer — evaluasi dulu baru implement. Boleh push back kalau salah.
/avcn-debug DEBUGsystematic-debugging WAJIB sebelum propose fix. 4 fase: Root Cause→Pattern→Hipotesis→Fix
/avcn-verify DEBUGverification-before-completion WAJIB sebelum bilang selesai — run command, lihat output, baru claim done.
/avcn-write-skill METAwriting-skills Buat skill baru yang bisa di-reuse — TDD approach, test dulu baru tulis skill.
