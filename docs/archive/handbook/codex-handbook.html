01

Overview — Apa itu Codex Skill?

v2026

Skill Codex = paket folder berisi instruksi + aset pendukung yang Codex/Cursor index dan munculkan di skills picker. Beda dengan slash command bawaan, skill dipanggil eksplisit dengan prefix $ (contoh: $classy-plan, $pdf) atau auto-trigger jika allow_implicit_invocation: true.

ℹ
Home directory skills: Semua skills disimpan di ~/.codex/skills/ (atau path custom via env var $CODEX_HOME). Nama folder = nama trigger. Contoh: folder classy-plan/ = trigger $classy-plan.
Workflow / Disiplin
classy-* Skills
Skills yang menset mode kerja. Wajib dipanggil eksplisit (allow_implicit_invocation: false). Contoh: brainstorming → plan → implement → verify → finish.
Technical / Domain
Domain Skills
Skills teknis dengan dependency, command, dan quality gate. Boleh auto-trigger via implicit keywords. Contoh: $pdf, $classy-playwright, $imagegen.
System / Meta
.system/* Skills
Built-in infrastructure skills. $openai-docs untuk latest model docs, $imagegen image generation, $plugin-creator untuk buat plugin baru.
Anatomi folder skill standar 2026
classy-plan/               ← kebab-case = nama trigger $classy-plan
├── SKILL.md               ← WAJIB: entry point + YAML frontmatter
├── agents/
│   └── openai.yaml        ← OPSIONAL: UI picker manifest (display_name, icon)
├── references/            ← OPSIONAL: detail on-demand (tokens.md, rules.md)
│   └── *.md
├── scripts/               ← OPSIONAL: eksekutabel ($CODEX_HOME/scripts/)
│   └── *.sh / *.py
└── assets/                ← OPSIONAL: ikon, gambar (./assets/icon.png)
02

Best Practices Codex 2026

10 Rules

Diekstrak dari ~/.codex/skills/CATATAN-FORMAT-SKILL-CODEX-2026.md — hasil deep-dive semua skills aktif per 2026-05-04.

01
Skill = Folder + SKILL.md + Frontmatter

Entry point selalu SKILL.md di root folder skill. Frontmatter YAML minimal butuh dua field:

---
name: classy-plan
description: Use when the user types /classy-plan or ...
---

Field name harus identik dengan nama folder. Validator: quick_validate.py — hanya izinkan name, description, license, allowed-tools, metadata.

02
Description = AI Matcher, Bukan Marketing Copy

Description dibaca AI matcher untuk discovery. Harus berisi:

  • → Trigger eksplisit (/skill-name atau $skill-name)
  • → Kondisi implicit (kata kunci, konteks)
  • → Negative trigger ("Skip when: …")
  • → Domain spesifik jika relevan
description: Use when the user types /classy-plan
  or wants to write a step-by-step plan. Skip when
  implementation is already underway.
03
Progressive Disclosure — SKILL.md ≤ 200 Baris

SKILL.md harus tipis. Detail dipindah ke references/ agar tidak membanjiri context window AI.

Short-form
≤25 baris — skill disiplin ringan (brainstorming, plan, implement)
Long-form
100–200 baris — skill teknis dengan commands dan examples
Long-form + ref
SKILL.md jadi index, detail di references/*.md — bentuk paling matang
04
agents/openai.yaml = UI Picker Manifest

File ini bikin skill tampil rapi di Codex skills picker:

interface:
  display_name: "Classy: Plan"
  short_description: "Write HANDOFF.md plan"
  brand_color: "#111111"
  icon_large: "./assets/icon.png"
  default_prompt: "Use $classy-plan to..."

policy:
  allow_implicit_invocation: false
05
allow_implicit_invocation — Set dengan Sengaja
false
Skill disiplin/proses — brainstorming, plan, implement, verify. Harus dipanggil eksplisit agar tidak nyala sembarangan.
true
Skill teknis — pdf, playwright, imagegen. Boleh auto-trigger via keywords di description.
⚠
JANGAN set allow_implicit_invocation: true pada skill destructive atau irreversible.
06
$CODEX_HOME — Jangan Hardcode Path

Semua path internal pakai env var, bukan absolute path. Ini memastikan skill portable di semua machine.

❌ JANGAN:
PWCLI="C:/Users/claud/.codex/skills/playwright/scripts/..."

✅ BENAR:
export CODEX_HOME="${CODEX_HOME:-$HOME/.codex}"
PWCLI="$CODEX_HOME/skills/playwright/scripts/playwright_cli.sh"
07
Quality Gate = Checklist [ ], Bukan Prosa

Quality gate harus berbentuk checklist agar AI bisa track completion state secara eksplisit.

❌ JANGAN (prosa):
"Make sure the output is complete and tested."

✅ BENAR (checklist):
## Quality Gate
- [ ] Real objective stated in one sentence
- [ ] At least one assumption named explicitly
- [ ] Single recommendation given, not "it depends"
- [ ] Next skill / step is named
08
Section "What to NEVER do" — Wajib di Governance Skill

Tanpa larangan eksplisit, skill mudah di-override. Format wajib:

## What to NEVER do
- ❌ Open with a 5-paragraph essay before answering
- ❌ Present 5+ options (decision paralysis)
- ❌ Refuse to recommend ("you decide")
- ❌ Slide into implementation without $classy-plan

Setiap larangan = loophole yang ditutup.

09
Naming — kebab-case, Frontmatter name = Folder Name
Folder name
classy-brainstorming — kebab-case murni, jadi trigger $classy-brainstorming
Frontmatter
name: classy-brainstorming — HARUS identik dengan folder
Prefix konvensi
classy-* untuk Sentra/Chief workflow, bare name untuk skill umum
⚠
Dash ganda (foo--bar) DILARANG — quick_validate.py reject regex. Single-dash only.
10
Long-form Skill Wajib Punya References Index Tabel

Skill dengan banyak detail wajib sediakan tabel index di SKILL.md agar AI tahu kapan load file mana:

## Reference Files Index
| File                    | Contents             | When to load        |
|-------------------------|----------------------|---------------------|
| `references/tokens.md`  | Complete CSS tokens  | Always for CSS gen  |
| `references/rules.md`   | 6 Golden Rules + CSS | For component gen   |
| `references/components.md`| 10 component patterns | Per component type|

Ini mencegah AI memuat semua file sekaligus dan membuang context window.

Writing Skills — TDD Approach

RED → GREEN → REFACTOR

Writing skills IS Test-Driven Development applied to process documentation. Jika tidak ada failing test dulu, Anda tidak tahu apakah skill mengajarkan hal yang benar.

⚠
Iron Law: NO SKILL WITHOUT A FAILING TEST FIRST. Tulis skill sebelum test? Delete it. Start over. No exceptions — tidak untuk "simple additions", tidak untuk "documentation updates".
Phase 1 · RED
Baseline Test — Watch It Fail

Run pressure scenario dengan subagent WITHOUT skill. Document exact behavior:

  • Apa choices yang agent buat?
  • Rationalizations apa yang digunakan (verbatim)?
  • Pressure mana yang trigger violations?
  • Loophole apa yang ditemukan?
Phase 2 · GREEN
Write Minimal SKILL.md

Tulis skill yang address rationalizations spesifik dari baseline. Jangan tambah content untuk hypothetical cases.

  • Frontmatter: name + description (triggers)
  • Overview 1–2 kalimat
  • Workflow numbered steps
  • Quality gate checklist [ ]
  • Section "What to NEVER do"
  • Run same scenarios WITH skill → verify compliance
Phase 3 · REFACTOR
Close Loopholes

Agent found new rationalization? Tambah explicit counter. Re-test sampai bulletproof.

  • Build rationalization table dari semua test iterations
  • Create Red Flags list untuk self-check
  • Address "spirit vs letter" arguments
  • Forbidden workarounds ditulis eksplisit
  • Re-test dengan pressure scenarios baru
Rationalization Table (template)
Excuse yang Sering DigunakanReality (jawaban yang harus ada di skill)
"Too simple to test"Simple code breaks. Test takes 30 seconds. Delete → start over.
"I'll test after"Tests-after = "what does this do?" Tests-first = "what should this do?" — beda fundamental.
"Tests achieve same goals anyway"Tests-after prove nothing. The Iron Law stands.
"It's about spirit not ritual"Violating the letter IS violating the spirit.
"This is different because..."Delete code. Start over. No exceptions.
Deployment Checklist — MANDATORY per skill
RED Phase
  • Create 3+ pressure scenarios
  • Run WITHOUT skill — document baseline verbatim
  • Identify patterns in rationalizations/failures
GREEN Phase
  • Name: letters, numbers, hyphens only
  • YAML frontmatter: name + description ≤ 1024 chars
  • Description starts with "Use when…"
  • Keywords for search (errors, symptoms, tools)
  • Run WITH skill → verify compliance

Tips & Tricks — Memaksimalkan Codex

Power User
Tip 1
$ vs / — Bedanya Invocation Prefix
$skill-name
Codex native prefix. Langsung trigger skill dari skills picker atau chat.
/skill-name
Claude Code slash command style. Bisa di-register via settings jika skill sudah diinstall di Claude.
Auto-trigger
Skill dengan allow_implicit_invocation: true auto-aktif jika description keywords match context.
Tip 2
Progressive Disclosure — Jaga Context Window

Context window AI adalah resource yang terbatas. Strategi:

  • → SKILL.md tipis (≤200 baris) — selalu di-load
  • → references/*.md — on-demand, hanya jika dibutuhkan
  • → Gunakan tabel References Index di akhir SKILL.md
  • → API docs 100+ baris → pisahkan ke file sendiri
Tip 3
Description CSO — Claude Search Optimization

Description harus kaya keyword agar AI bisa find skill saat dibutuhkan:

description: Use when the user types /classy-plan
  or wants to write an implementation plan,
  HANDOFF.md, step-by-step task breakdown,
  rollback strategy.
  # ↑ keyword: plan, HANDOFF, rollback, steps
  Skip when: implementation is underway.

JANGAN summarize workflow di description — AI akan pakai description sebagai shortcut dan skip body skill.

Tip 4
Skill Chaining — Feeds Pattern

Setiap workflow skill sebaiknya eksplisit menyebut skill berikutnya di akhir body:

## Pairs with
- Feeds → $classy-plan once direction is chosen
- Replaces → ad-hoc "let me think" without structure

Ini membantu AI tahu next step tanpa perlu explicit instruction dari Chief.

Tip 5
$CODEX_HOME — Portable Skills

Setup env var di shell profile untuk portabilitas:

export CODEX_HOME="${CODEX_HOME:-$HOME/.codex}"
# Kemudian di scripts skill:
PWCLI="$CODEX_HOME/skills/classy-playwright/scripts/playwright_cli.sh"

Custom location: export CODEX_HOME="D:/DevOps/.codex" jika skills disimpan di tempat lain.

Tip 6
Composio — 1000+ Apps tanpa Custom Integration

Gunakan $composio untuk connect external apps tanpa tulis custom integration:

CLI mode
Langsung dari terminal, cocok untuk satu-off tasks
SDK mode
Build persistent AI agent dengan tool access (Gmail, GitHub, Notion, Slack)
MCP mode
Connect via MCP server untuk Claude Code integration
Tip 7
Quality Gate Format — Checklist Bukan Prosa

AI dapat track state completion dari checklist [ ]. Prosa tidak memberikan struktur yang jelas.

❌ Prosa (tidak trackable):
"Ensure the code is complete and well-tested."

✅ Checklist (trackable):
- [ ] Tests pass: npm test
- [ ] Build success: npm run build
- [ ] No TypeScript errors: tsc --noEmit
- [ ] Diff verifiable: git diff
Tip 8

command -v tool + install instructions jika missing
Workflow

Anti-Patterns — Yang Harus Dihindari

❌ 9 Violations

Diekstrak dari CATATAN section 9. Setiap anti-pattern menyebabkan skill tidak discoverable, tidak maintainable, atau berbahaya digunakan.

❌ Anti-Pattern✅ Correct Approach
SKILL.md > 300 baris tanpa references/ Pecah ke references/*.md. SKILL.md harus jadi index tipis ≤200 baris. Content heavy → on-demand references.
Description generik: "helps with files", "for coding tasks" Description adalah AI matcher. Tulis trigger eksplisit (/skill-name) + kondisi implicit + domain spesifik + negative trigger.
Hardcode absolute path: "C:/Users/claud/.codex/..." Selalu pakai $CODEX_HOME env var. Membuat skill portable lintas machine.
Trigger ambigu: skill pdf vs make-pdf vs pdf-tool dalam satu repo Gunakan satu trigger name yang jelas dan konsisten. Cek konflik sebelum bikin skill baru.
Tidak ada section "What to NEVER do" Skill tanpa larangan eksplisit mudah di-rationalize. Setiap skill governance wajib punya ❌ list.
allow_implicit_invocation: true pada skill destructive/irreversible Skill yang bisa hapus data, push code, atau send message wajib false — harus dipanggil eksplisit.
Campur instruksi human-doc dan agent-doc dalam satu file Split: SKILL.md untuk human-readable, AGENTS.md untuk auto-generated agent context (composio pattern).
Skill tanpa prerequisite check, tapi asumsikan tool eksternal terinstall Selalu tambah check di awal body: command -v npx >/dev/null || echo "install npx".
Quality gate berbentuk prosa (tidak trackable oleh AI) Format wajib: checklist [ ] dengan item konkret dan verifiable (npm test, tsc --noEmit, dll.).
✓
Quick Checklist — Sebelum Deploy Skill Baru:
  • ☐ Folder kebab-case di ~/.codex/skills/<name>/
  • ☐ SKILL.md ada, frontmatter name + description
  • ☐ Description: trigger eksplisit + use-case + negative trigger
  • ☐ Body: Invocation → Workflow → Output style → NEVER → References
  • ☐ SKILL.md ≤ 200 baris (kalau lebih, pecah ke references/)
  • ☐ agents/openai.yaml ada kalau skill tampil di picker
  • ☐ allow_implicit_invocation di-set dengan sengaja
  • ☐ Dependency check di awal body (command -v X + install instructions)
  • ☐ Quality gate berbentuk checklist [ ]
  • ☐ Path internal pakai $CODEX_HOME, bukan hardcode
  • ☐ Baseline test (RED) sudah dijalankan sebelum menulis skill
