Rules & hierarki (cursor.com/docs)
§ Hierarki & benturan
Urutan umum saat aturan aktif: Team Rules → Project Rules (.cursor/rules/) → User Rules (Settings). Jika konflik, sumber yang lebih tinggi biasanya menang.

File legacy .cursorrules masih didukung; untuk repo baru disarankan migrasi ke .cursor/rules/*.mdc agar bisa dipecah per domain dan punya metadata aktivasi.
◈ Empat mode aktivasi
AlwaysalwaysApply: true — selalu dimasukkan konteks.
Per fileglobs — otomatis saat file cocok direferensikan.
Agentdescription — Agent memutuskan relevansi (Apply Intelligently).
ManualTanpa globs + tanpa always — hanya lewat @nama-rule.
Satu rule = satu concern; deskripsi jelas untuk mode Agent
Megarule ratusan baris tanpa glob (boros konteks)
A AGENTS.md vs .mdc
AGENTS.md di root (atau subfolder): Markdown polos, cocok untuk kontrak singkat tim.

.cursor/rules/*.mdc: frontmatter YAML + isi — cocok saat perlu kontrol kapan rule ikut (glob / deskripsi / always / manual).

Perintah bantu: /create-rule dari Agent untuk scaffold file + frontmatter.
T Team / Enterprise
Admin dapat memaksa rule organisasi dari dashboard Cursor. Team Rules bisa berupa teks bebas + opsi glob; perilaku pastikan di dokumentasi versi Cursor yang dipakai tim.
Context, @, indexing
@ Referensi konteks
Gunakan @file, @folder, simbol codebase/docs sesuai UI Cursor untuk menunjuk sumber tegas. Hindari menarik folder besar yang tidak relevan — hemat token dan mengurangi noise retrieval.
⊘ .cursorignore
Sintaks seperti .gitignore. Pola di sini membatasi akses fitur AI (Tab, Agent, @) dan indexing semantic — berguna untuk rahasia, artefak build, vendor besar.

Batasan: terminal Agent dan tool MCP tidak dijamin terblokir oleh .cursorignore — jangan mengandalkan ignore sebagai satu-satunya kontrol keamanan untuk rahasia runtime.
◎ .cursorindexingignore
Hanya mengecualikan dari indexing / codebase search; file bisa tetap dibaca Agent bila diminta eksplisit. Cocok untuk generated besar yang mengganggu pencarian tapi kadang perlu dibuka.
↻ Index & monorepo
Setelah mengubah ignore: refresh lewat Settings → Features → Codebase Indexing. Opsional: Hierarchical Cursor Ignore untuk membaca .cursorignore di parent directory (besar monorepo).
Model Context Protocol
◇ Konfigurasi
.cursor/mcp.json — per proyek (version control).
~/.cursor/mcp.json — global mesin pengembang.

Cursor mendukung variabel di field seperti command, args, env, url, headers untuk kredensial tanpa hardcode di repo publik.
⇄ Transport & kemampuan
Umum: stdio (proses lokal) dan HTTP/streamable untuk server remote / tim.

Protokol: Tools, Prompts, Resources; ekstensi Roots, Elicitation, MCP Apps (UI interaktif; host lama tetap dapat respons teks).
⚡ Pakai aman di Agent
Secara default Agent meminta persetujuan sebelum menjalankan tool MCP — tinjau argumen di UI.

Auto-run (seperti terminal) bisa diaktifkan dengan hati-hati. Pre-konfigurasi tool yang boleh auto-run: ~/.cursor/permissions.json (sesuai dokumentasi versi).

OAuth: marketplace / plugin; untuk provider tertentu bisa set kredensial OAuth statis di mcp.json bila didukung.
✓ Praktik
Pasang server dari Cursor Marketplace bila tersedia; uji di project kecil dulu
JSON mcp.json rusak (satu koma) memutus seluruh server
Setelah edit config: restart Cursor bila server tidak muncul di Settings → Tools & MCP.
Agent Skills (SKILL.md)
◇ Lokasi discovery
Proyek: .cursor/skills/, .agents/skills/. Global: ~/.cursor/skills/, ~/.agents/skills/. Kompatibilitas: juga memuat dari .claude/skills/, .codex/skills/ dan homolog global.

Struktur: folder per skill berisi SKILL.md; opsional scripts/, references/, assets/.
✎ Frontmatter wajib
name — identifikator; huruf kecil, angka, hyphen; harus sama dengan nama folder induk.
description — kapan skill dipakai; dipakai Agent untuk relevansi.
Opsional: disable-model-invocation: true → hanya lewat /nama-skill atau @, tidak auto-detect. --- name: deploy-staging description: Deploy ke staging setelah test hijau. ---
↔ Rules vs Skills
RulesKonvensi ringkas, sering always / glob — biaya konteks tetap.
SkillsWorkflow multi-langkah; konteks penuh saat relevan / dipanggil.
Detail panjang taruh di references/; SKILL.md tetap fokus
Duplikasi besar antara rule always-on dan skill
⌁ Migrasi
/create-skill — wizard skill baru.
/migrate-to-skills (Cursor 2.4+): mengonversi rule “Apply Intelligently” (tanpa globs) dan slash command menjadi skill; rule alwaysApply: true atau ber-globs tidak ikut migrasi otomatis.
Agent, Plan, izin
▣ Plan Mode
Untuk refactor besar atau banyak file: aktifkan perencanaan sebelum eksekusi agar langkah terlihat dan bisa dikoreksi awal. Plan Mode tetap dapat memakai MCP bila tool relevan.
! Persetujuan & auto-run
Tinjau diff terminal dan tool sebelum menyetujui batch berisiko. Auto-run menghemat klik tetapi meningkatkan radius kerusakan — cocok hanya untuk workspace terpercaya dan tool set yang sempit.
∞ Extension API (MCP)
Untuk setup enterprise / otomasi: registrasi server MCP programatis (mis. vscode.cursor.mcp.registerServer()) agar tidak mengubah file mcp.json langsung — cek referensi API versi Cursor yang terpasang.
✓ Checklist singkat
Repo: rule terpisah + skill untuk prosedur berulang
Menyimpan secret API di rule/skill yang di-commit
Uji MCP dengan satu tool dulu; monitor Output → Hooks bila ada
Mengandalkan .cursorignore sebagai satu-satunya boundary keamanan
Hooks (agent lifecycle)
⌗ Lokasi file
Proyek: .cursor/hooks.json (repo root). User: ~/.cursor/hooks.json. Cursor memuat ulang saat file disimpan; jika tidak jalan, reload window / restart app.

Setiap event memetakan ke array definisi hook — beberapa perintah bisa dijalankan berurutan.
⚙ Bentuk minimal
{ "version": 1, "hooks": { "beforeSubmitPrompt": [{ "command": ".cursor/hooks/guard.sh" }], "afterFileEdit": [{ "command": ".cursor/hooks/format.sh" }] } } command bisa string shell, path absolut, atau relatif (CWD tergantung sumber hook — sesuaikan path proyek).
↺ Event Agent vs Tab
Agent / Cmd+K: antara lain sessionStart, sessionEnd, preToolUse, postToolUse, subagentStart/subagentStop, beforeShellExecution/afterShellExecution, beforeMCPExecution/afterMCPExecution, beforeReadFile, afterFileEdit, beforeSubmitPrompt, preCompact, stop, afterAgentResponse, afterAgentThought.

Tab inline: beforeTabFileRead, afterTabFileEdit.
✓ Praktik
Skrip harus executable. Gunakan matcher pada hook generik (mis. preToolUse) untuk membatasi ke Shell|Read|Write.

Hook = observasi / enrich / veto di pipeline; sandbox terminal adalah lapisan lain — kombinasikan untuk defense in depth.
