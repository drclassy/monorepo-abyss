07 · Command Examples

Short usage examples so the team can invoke the right mode without drifting.
CommandUse whenActions
/classy-usingThe task just started and the correct mode is not yet explicit.
/classy-execution-controlThe task is non-trivial and needs strict scope boundaries first.
/classy-investigateThe code owner, architecture path, or likely edit surface is not yet clear.
/classy-planThe task needs sequencing before code changes begin.
/classy-implementThe direction is already clear and the task should move into execution.
/classy-debugThere is a failure, but the cause is not yet understood.
/classy-reviewThe task is evaluation-first and should focus on findings.
/classy-refactorThe goal is structural improvement without intended behavior change.
/classy-verifyThe work needs evidence before it can be considered complete.
/classy-finishThe task is ready to close with an evidence-based summary.
