Context Management
⟳ Kapan /clear vs /compact
/clear — ganti task yang tidak berhubungan, dikerjakan 2x+ di satu session, atau context sudah kacau.

/compact focus on X — masih di task yang sama tapi context hampir penuh (~50%). Tambahkan focus keyword biar summary tepat sasaran.
/compact focus on the auth refactor
/compact
@ @-Mention yang Efisien
Jangan embed file besar ke prompt. Pakai pointer dulu, baru minta Claude baca kalau perlu.

@.agent/HANDOFF.md — muat file spesifik
@src/services/ — muat seluruh folder
@AGENTS.md §5 — referensi section tertentu

Hindari @node_modules atau file besar yang irrelevant — buang context sia-sia.
↩ /rewind & Checkpoints
double-Esc atau /rewind — undo last change.
3 mode: code only, conversation only, atau keduanya.

Catatan penting: /rewind track file edits Claude saja. Bash rm, mv, cp tidak ter-capture. Selalu stash dulu sebelum risky run:
# sebelum task berisiko git stash push -u -m "pre-task"
⑂ Session Branching
/fork-session — clone session dengan ID baru. Cocok untuk explore dua pendekatan berbeda tanpa buang progress.

claude -c — lanjut session terakhir
claude --resume — pilih session dari list
claude --from-pr 42 — resume session yang terkait PR #42

Pro tip: Selalu /handoff di akhir session biar HANDOFF.md up-to-date sebelum tutup.

⟳ Thinking Keywords
Keyword ini adalah soft cues di atas effort level — pakai dengan hemat, bukan di setiap prompt.

◈ Scope yang Jelas
Selalu sebut scope eksplisit — file path, function name, line range. Semakin narrow scope-nya, semakin akurat hasilnya. # Explicit scope — bagus Refactor PatientService.create() di apps/healthcare/sentra-main/src/ patient/patient.service.ts:L24-L67 # tanpa ganti struktur yang lain
⚡ Parallelism Explicit
Opus 4.7 spawn lebih sedikit subagent secara default dibanding 4.6. Kalau butuh parallel, minta secara eksplisit. # Minta paralel secara eksplisit Spawn one subagent per module in parallel: - sentra-main auth module - sentra-assist referral module - packages/database migrations

Jangan Dilakukan — Common Traps
✕ CLAUDE.md Overloading
Masukin formatting rules (pakai .eslintrc/prettier)
Stack facts, commands, hard constraints, @path pointers
CLAUDE.md > 200 baris (rule adherence turun)
Target < 50 baris per project, sisanya @pointer
"Double-check before returning" (4.7 sudah otomatis)
Tulis hanya yang Claude salah tanpa instruksi
✕ Context & Session Traps
Tunggu auto-compact — bisa fire di tengah eksekusi
/compact manual di ~50% sebelum task kritis
Langsung lanjut task baru di session yang sama tanpa /clear
/clear dulu — context lama bisa kontaminasi task baru
Rely pada memory antar session tanpa .agent/ files
/handoff di akhir setiap session — HANDOFF.md jadi sumber kebenaran
✕ Agentic & Planning Traps
Langsung kode tanpa brainstorm/write-plan untuk task besar
/avcn-brainstorm → /avcn-write-plan → GO → /avcn-execute-plan
Anggap "ok" atau emoji = GO untuk Class C task
Literal token "GO" satu baris sendiri — tidak ada shortcut
Biarkan agent fix 3+ kali tanpa intervensi
Max 2 autonomous fix — stop & /avcn-debug pada attempt ke-3
✕ Permissions & Hooks Traps
Exit code 1 di hook — Claude Code abaikan silently
Exit code 2 untuk block + feed error ke Claude sebagai context
Hanya pakai deny array untuk proteksi destructive commands
Double layer: deny array + PreToolUse hook (deny menang, tapi hook lebih informative)
--dangerously-skip-permissions di project settings
CLI flag saja, jangan commit — project settings ignore flag ini
