Tips & Tricks

Practical Codex habits

These are the habits that kept ABYSS safe during cleanup, merge conflict resolution, and push.

1. Keep AGENTS.md thin
  • Put durable rules in AGENTS.md.
  • Put detailed routers, reports, and playbooks in reference docs or skills.
  • Never turn the global baseline into a giant handbook.
2. Treat dirty trees as evidence, not chaos
  • First classify changes: governance, config, workspace, prototype, session history, unknown.
  • Restore accidental deletions before committing anything.
  • Commit one coherent scope at a time.
3. Worktree and parallel mode are not defaults
  • Use worktrees only when isolation clearly helps.
  • Use parallel agents only when files, architecture decisions, dependencies, and verification do not overlap.
  • If overlap is real, stay serial.
4. Lockfile conflicts need special handling
  • Audit remote-only range before merging.
  • Prefer merge over rebase when local history is already meaningful and should not be rewritten.
  • For pnpm lock conflicts, resolve narrowly, regenerate lockfile, then run frozen-lockfile verification.
5. No “done” without fresh proof
  • Use the smallest relevant verification: targeted test, lint, typecheck, build, or behavior check.
  • If verification cannot run, report exactly why and what risk remains.
  • For engine-like work, test multiple inputs and prove outputs vary correctly.
6. Audit automation before edit automation
  • Recurring automations should start as review-only.
  • Never let automation do destructive git or repo changes without explicit scope.
  • Use DCS audit skills to find drift before changing AGENTS.md or skills.
Operating Rules

Safety enforcement

The goal is not more ceremony. The goal is fewer agent mistakes, cleaner commits, and verified progress.

Do
  • Read SSOT before action.
  • Write brief notes before implementation.
  • Make the smallest complete change.
  • Verify with evidence before reporting completion.
  • Use scoped commits and clean push-readiness audits.
Be careful with
  • pnpm-lock.yaml, pnpm-workspace.yaml, MCP config, hooks, and package manifests.
  • Repo-local vs global Codex setup.
  • Untracked session/history files.
  • Prototype files with local absolute paths or unused assets.
Do not allow
  • Force push, reset, clean, broad deletion, or rewrite history without explicit instruction.
  • Touching packages/sentra/** crown-jewel areas without explicit scope.
  • Claiming completion from appearance only.
  • Mixing unrelated changes into one commit.
Expected outcome Every important task should end with SSOT used, reference used, brief notes, scoped changes, verification evidence, checklist recheck, remaining risk, and one next step.
