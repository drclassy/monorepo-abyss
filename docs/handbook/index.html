05 CLAUDE.md Structure CRITICAL
⚠ Hard limit: keep CLAUDE.md under ~200 lines. Claude Code injects ~50 system instructions already. Every rule you add dilutes every other rule. Rule adherence degrades uniformly past ~200 total instructions.
What TO put in CLAUDE.md
Stack factsRuntime, package manager, build tool, test runner — things Claude gets wrong without them.
Key commandsDev, test, lint, migrate. Exact commands only.
Hard constraintsFiles that must never be touched, patterns that must be followed.
Workflow gatePlan-then-execute requirement, JET protocol reference.
@path pointersReferences to deeper docs. Loaded on demand, not bulk embedded.
What NOT to put in CLAUDE.md
Formatting rulesPut in .eslintrc / .prettierrc. Linters enforce, Claude forgets.
Already-obvious thingsIf Claude picks up pnpm from lockfile, don't restate it.
Self-verify remindersOpus 4.7 self-verifies natively — telling it to re-check wastes tokens and triggers overthinking.
Status reminders"Print status every N steps" — Opus 4.7 does this natively.
Hook-enforceable rulesIf a PreToolUse hook blocks it, don't waste a CLAUDE.md line.
Minimal template (target: <50 lines per project)

07 Permission System SECURITY
⛔ Evaluation order: deny → hooks → allow → defaultMode. Deny always wins — even inside bypassPermissions. Use both deny array AND PreToolUse hooks for double-layer protection on destructive commands.
Syntax patterns
"permissions": {
  "defaultMode": "default",  // or "plan" or "bypassPermissions"
  "allow": [
    "Bash(git:*)",          // wildcard on subcommands
    "Bash(pnpm:*)",
    "Write(src/**)",         // glob on file path
    "Edit(tests/**)",
    "mcp__github__*"         // MCP tool: mcp__server__tool
  ],
  "ask": [
    "Bash(gh pr merge:*)",   // prompt before merging
    "Write(packages/db/**)"  // prompt before db changes
  ],
  "deny": [
    "Bash(rm -rf:*)",        // HARD BLOCK
    "Bash(git push --force:*)",
    "Bash(git reset --hard:*)",
    "Bash(terraform apply:*)",
    "Read(.env)",
    "Read(.env.*)",
    "Read(secrets/**)"
  ]
}
Built-in tool names
ToolPattern
BashBash(cmd:*) or Bash(cmd subcommand:*)
ReadRead(path/glob/**)
WriteWrite(path/glob/**)
EditEdit(path/glob/**)
MultiEditMultiEdit(path/**)
MCP toolsmcp__servername__toolname
⚠ --dangerously-skip-permissions is ignored when set inside project-level settings — only works in CLI. Never commit this flag.
08 Hooks System AUTOMATION
Hooks run shell commands at lifecycle events. Exit code 2 is special — it blocks the action and feeds stderr back to Claude as context. Exit 1 is silently ignored. Exit 0 proceeds normally.
PreToolUse Runs BEFORE any tool call — can block with exit 2 BLOCKING 
# Block destructive commands
{
  "matcher": "Bash",
  "hooks": [{
    "type": "command",
    "command": "pwsh -Command \"
  $cmd = $env:CLAUDE_BASH_COMMAND;
  $blocked = @('rm -rf','git reset --hard',
               'terraform apply','DROP TABLE');
  foreach ($b in $blocked) {
    if ($cmd -match [regex]::Escape($b)) {
      Write-Error ('BLOCKED: ' + $b);
      exit 2
    }
  }\""
  }]
}
PostToolUse Runs AFTER tool call — good for auto-verify NON-BLOCKING 
# Auto-test after file changes
{
  "matcher": "Write|Edit|MultiEdit",
  "hooks": [{
    "type": "command",
    "command": ".claude/hooks/verify.ps1"
  }]
}
# verify.ps1 content:
if (-not (pnpm test --silent 2>&1)) {
  git stash push -u -m "agent-failed"
  Write-Error "Tests failed — changes stashed"
  exit 2  # feeds error back to Claude
}
Stop Runs when Claude stops (end of response) LOGGING 
# Auto-write session log
{
  "hooks": [{
    "type": "command",
    "command": "pwsh -Command \"
  $d = Get-Date -Format 'yyyy-MM-dd HH:mm';
  $f = '.agent/sessions/' +
       (Get-Date -Format 'yyyy-MM-dd') + '.md';
  if (!(Test-Path (Split-Path $f))) {
    New-Item -Type Directory (Split-Path $f) -Force
  }
  Add-Content $f ('## Session end: ' + $d)\""
  }]
}
SessionStart Runs on startup/clear/compact — context injection CONTEXT 
# GUARD 1 — verify .agent/ files exist
{
  "matcher": "startup|clear|compact",
  "hooks": [{
    "type": "command",
    "command": "pwsh -Command \"
  $files = @('.agent/CONTEXT.md',
    '.agent/PROGRESS.md',
    '.agent/HANDOFF.md',
    '.agent/LESSONS.md',
    '.agent/DECISIONS.md');
  $missing = $files | Where-Object {!(Test-Path $_)};
  if ($missing) {
    Write-Error ('GUARD 1 FAILED: ' +
      ($missing -join ', '));
    exit 2
  }\""
  }]
}
ℹ Hooks are defined in .claude/settings.json under "hooks". PreToolUse and PostToolUse hooks receive the tool name via CLAUDE_TOOL_NAME env var. Bash hooks get CLAUDE_BASH_COMMAND. Add "async": true for non-blocking hooks that don't need to gate execution.

13 JET Protocol Quick Reference WORKFLOW
9-phase workflow
J1
Context Gather
Scan .agent/, repo, env vars. Log confirmation of full scan.
AUTO
J2
Validate
Check plan vs .cursor/rules/ + AGENTS.md. Halt if CRITICAL violation.
AUTO
J3
Diagnose
Root cause analysis. Document in HANDOFF.md §Diagnosis.
AUTO
J4
Plan
Full HANDOFF.md: numbered steps, files, rollback, acceptance criteria.
AUTO
J5
⛔ WAIT FOR GO
Hard gate. Zero execution until Chief types the literal token "GO" in current session. Prior GO does not carry over. "ok", emoji, silence = NOT approval.
MANUAL — CLASS C
J6
Execute
Implement exactly per HANDOFF.md. Diff verifiable via git diff. Nothing outside plan scope.
POST-GO
J7
Verify
100% tests pass or rollback. Max 1 autonomous fix per failure. ERROR LOOP DETECTED on 2nd failure.
POST-GO
J8
Documentation
Update .agent/sessions/ + HANDOFF.md with current state and next action.
POST-GO
J9
Commit
Git commit with trailers: Agent: Claude · Phase: Execution · Handoff: [id]
POST-GO
Task classification
Class A
Minimal risk — Read, grep, typo fix, rename.
Phases: J1–J4 only. Auto-approve.
Class B
Standard — Component, API, bug fix, refactor.
Phases: J1–J7. Checkpoint mode (log plan, proceed).
Class C
High risk — DB migration, infra, PHI, polyrepo.
Phases: J1–J9 full. Hard GO gate at J5.

Cursor AI + Claude Code combo
Use them for different things. Cursor AI (Ctrl+L) is great for quick inline completions and single-file edits. Claude Code handles multi-file agentic work, planning, test runs, and git operations. Don't duplicate effort — if Claude Code is mid-task, don't use Cursor AI to edit the same files.
Windows-specific (PowerShell hooks)
On Windows, always use pwsh (PowerShell 7) not bash in hooks — it's more reliable in Cursor's integrated terminal. Use the Filesystem MCP tools through Claude Code for file operations rather than raw bash.
# Correct: PowerShell in hooks
"command": "pwsh -Command \"...\""

# If bash is needed (Git Bash)
"command": "\"C:\\Program Files\\Git\\bin\\bash.exe\" -c \"...\""
