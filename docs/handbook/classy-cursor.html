Rules & Hierarki (cursor.com/docs)
1 Hierarki & Benturan
Urutan: Team Rules - Project Rules (.cursor/rules/) - User Rules (Settings). Konflik: sumber lebih tinggi biasanya menang. File legacy .cursorrules masih didukung; disarankan migrasi ke .cursor/rules/*.mdc agar dipecah per domain dengan metadata aktivasi.
2 Empat Mode Aktivasi
AlwaysalwaysApply: true - selalu dimasukkan konteks.
Per fileglobs - otomatis saat file cocok.
Agentdescription - Agent memutuskan relevansi.
ManualTanpa globs + tanpa always - hanya @nama-rule.
Satu rule = satu concern; deskripsi jelas untuk mode Agent
Megarule ratusan baris tanpa glob
3 AGENTS.md vs .mdc
AGENTS.md di root (atau subfolder): Markdown polos, cocok untuk kontrak singkat tim. .cursor/rules/*.mdc: frontmatter YAML + isi. Perintah: /create-rule dari Agent untuk scaffold.
4 Team / Enterprise
Admin dapat memaksa rule organisasi dari dashboard Cursor. Team Rules bisa teks bebas + opsi glob; pastikan di dokumentasi versi Cursor yang dipakai tim.
Context, @, Indexing
@ Referensi Konteks
Gunakan @file, @folder, simbol codebase/docs sesuai UI Cursor. Hindari menarik folder besar yang tidak relevan -- hemat token.
! .cursorignore
Sintaks seperti .gitignore. Membatasi akses fitur AI (Tab, Agent, @) dan indexing semantic. Batasan: terminal Agent dan tool MCP tidak dijamin terblokir oleh .cursorignore.
~ .cursorindexingignore
Hanya mengecualikan dari indexing / codebase search; file bisa tetap dibaca Agent bila diminta eksplisit. Cocok untuk generated besar.
* Index & Monorepo
Refresh: Settings > Features > Codebase Indexing. Opsional: Hierarchical Cursor Ignore untuk parent directory di monorepo besar.
Model Context Protocol
M Konfigurasi
.cursor/mcp.json - per proyek. ~/.cursor/mcp.json - global. Cursor mendukung variabel di field command, args, env, url, headers.
T Transport & Kemampuan
stdio (proses lokal) dan HTTP/streamable untuk server remote. Protokol: Tools, Prompts, Resources.
A Pakai Aman di Agent
Default: Agent minta persetujuan sebelum MCP tool. Auto-run bisa diaktifkan dengan hati-hati via ~/.cursor/permissions.json. OAuth: marketplace / plugin.
+ Praktik
Pasang dari Marketplace; uji di project kecil dulu
JSON mcp.json rusak memutus seluruh server
Agent Skills (SKILL.md)
S Lokasi Discovery
Proyek: .cursor/skills/, .agents/skills/. Global: ~/.cursor/skills/, ~/.agents/skills/. Juga memuat dari .claude/skills/, .codex/skills/.
F Frontmatter Wajib
name - identifikator. description - kapan skill dipakai. Opsional: disable-model-invocation: true - hanya lewat perintah manual.
R Rules vs Skills
RulesKonvensi ringkas, sering always/glob.
SkillsWorkflow multi-langkah; konteks penuh saat relevan.
M Migrasi
/create-skill - wizard skill baru. /migrate-to-skills (Cursor 2.4+): konversi rule Apply Intelligently ke skill.
Agent, Plan, Izin
P Plan Mode
Untuk refactor besar atau banyak file: aktifkan perencanaan sebelum eksekusi agar langkah terlihat dan bisa dikoreksi awal.
! Persetujuan & Auto-run
Tinjau diff sebelum menyetujui. Auto-run cocok hanya untuk workspace terpercaya dan tool set sempit.
E Extension API (MCP)
Registrasi server MCP programatis via vscode.cursor.mcp.registerServer().
+ Checklist
Rule terpisah + skill untuk prosedur berulang
Menyimpan secret di rule/skill yang di-commit
Hooks (Agent Lifecycle)
H Lokasi File
Proyek: .cursor/hooks.json. User: ~/.cursor/hooks.json. Cursor memuat ulang saat file disimpan.
C Bentuk Minimal
version: 1, daftar event ke array definisi hook dengan command, matcher, timeout.
E Event Agent vs Tab
Agent: sessionStart/End, preToolUse, postToolUse, beforeShellExecution, dll. Tab: beforeTabFileRead, afterTabFileEdit.
+ Praktik
Skrip executable + matcher untuk batasi ke tool spesifik
Hook sebagai satu-satunya lapisan keamanan
Urutan Workflow Cursor — Best Practice 2026
1 Setup Awal (sekali doang)
Install Cursor, login Anysphere, aktifkan Privacy Mode di Settings > Privacy. Pilih default model (Opus 4.7 untuk reasoning, Sonnet 4.6 untuk speed). Set theme + font Nerd Font.
Privacy Mode ON sebelum buka repo healthcare
Auto-update Cursor saat sedang ada session aktif
2 Project Init & Rules
Buka repo. Paste User Rules global (~/.cursor/USER_RULES.md) ke Settings > Rules. Pastikan .cursor/rules/*.mdc repo ke-load otomatis. Konflik: User Rules + Project Rules digabung — taruh language lock di global.
Numerik prefix (00-, 10-, 20-) untuk load order
Megarule 500+ baris tanpa glob
3 Context Hygiene
Set .cursorignore (block AI access penuh) + .cursorindexingignore (skip index, masih bisa @ref). Trigger Codebase Indexing sampai 100%. Untuk monorepo besar: aktifkan Hierarchical Cursor Ignore.
Exclude apps yang tidak aktif untuk fokus model
Indeks node_modules atau .next
4 MCP & Plugins
Edit .mcp.json repo dan ~/.cursor/mcp.json global. Pasang minimal: context7 (docs lookup), github (issue/PR). Pakai env var untuk PAT/secret — bukan plain text. Aktifkan plugin Marketplace yang relevan saja.
PAT via ${'$'}{GITHUB_PERSONAL_ACCESS_TOKEN} env var
Commit .mcp.json berisi token plain
5 Skills, Commands, Notepads
Bootstrap reusable: .cursor/skills/ (workflow multi-langkah), .cursor/commands/ (slash command custom), .cursor/notepads/ (template untuk @notepad). Pakai /create-skill wizard untuk scaffold.
Satu skill = satu tugas spesifik
Hapal command — pakai dropdown / palette
6 Daily Loop — Plan > Compose > Diff
Mulai task: Plan Mode dulu untuk multi-file refactor. Composer: pakai @codebase awal, lalu @filename spesifik. Setiap diff: review sebelum accept. Loop > 3 turn buntu? Restart conversation dengan ringkasan baru.
Plan Mode untuk Class B/C work
Auto-accept all tanpa baca diff
7 Pre-Commit Gate
husky + lint-staged jalan otomatis sebelum commit: secret scan (PAT, API key), PHI scan (NIK, BPJS, phone), lint-fix file staged saja. Bypass darurat: --no-verify (tidak direkomendasikan). FAQ bootstrap: pnpm install --package-import-method=copy git config core.hooksPath .husky/_ pnpm exec lint-staged --version
Di volume V:, pakai --package-import-method=copy bila install gagal karena reflink / copy-on-write
Kalau hook tidak terpanggil, cek git config core.hooksPath harus menunjuk ke .husky/_
Test fixture pakai data synthetic (NIK 0000...)
Commit .env walau gitignored sekalipun
8 Self-Review & Healthcare Check
Sebelum PR: panggil /healthcare-review (PHI + audit + FHIR + auth). Run pnpm security:primary-healthcare. Pakai code-reviewer sub-agent untuk diff review independen. Gate: critical 0, warning < 3.
Sub-agent review sebelum minta human review
Skip healthcare-review untuk endpoint patient data
9 Handoff & Memory Update
Tulis session log .agent/sessions/YYYY-MM-DD.md. Update .agent/HANDOFF.md + PROGRESS.md. Commit dengan trailer: Agent: Claude · Phase: Execution · Handoff: <id>. Tutup window — hot exit preserve state.
Conventional Commits + scope (healthcare/orchestrator/dll)
Force push ke main / branch protected
+ Loop Mingguan
Audit .cursor/rules/ — hapus rule deprecated. Update .agent/LESSONS.md dari incident minggu lalu. Rotasi PAT & secret. Cek update Cursor + plugin (manual mode supaya tidak surprise). Backup config ke private dotfiles repo.
Rotasi PAT minimal tiap 90 hari
Auto-update saat sedang ada session aktif
