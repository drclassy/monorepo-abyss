Repo Constitution

Rules, folders, dan authority hierarchy

Jangan semua instruksi digandakan di semua tempat. Buat layer yang jelas.

AGENTS.md

Highest policy authority lintas agent: scope, safety, architecture, verification.

.clinerules/

Adapter khusus Cline: persona, workflow, guardrails, verification.

.clinerules/workflows/

Custom slash workflow. Filename menjadi command: /audit-before-push.md.

.agent/

Operational SSOT/state: current status, decisions, handoff, project memory.

.cursor/rules/

Guard khusus Cursor. Jangan jadikan authority tertinggi jika AGENTS.md ada.

.codex/

Layer perilaku Codex jika relevant. Hindari konflik wording dengan AGENTS.md.

.clineignore

Kurangi noise dan lindungi file sensitif: node_modules, dist, build, secrets, env.

YOLO Mode

Jangan aktif di repo utama. Boleh hanya sandbox disposable.

Best Practice 2026

Tips dan trik Cline paling penting

Ini adalah checklist operasional harian untuk memaksimalkan Cline tanpa membuat repo berantakan.

1. Plan Mode untuk yang belum jelas
  • Fitur baru, arsitektur, bug sulit, security review, codebase asing.
  • Act Mode hanya setelah plan dan file list jelas.
2. Model strategy
  • Default: GPT-5.4 Medium.
  • High untuk refactor besar/arsitektur/debugging sulit.
  • GPT-5.2 untuk edit kecil dan hemat.
3. Verification first
  • Jangan percaya narasi sukses.
  • Wajib ada build/typecheck/test/lint atau alasan mengapa tidak bisa.
4. Browser off by default
  • Untuk task lokal, matikan browser agar tool-call tidak melebar.
  • Nyalakan hanya jika perlu dokumen web atau UI testing.
5. Jaga layout IDE
  • YOLO off.
  • Lock editor group jika perlu.
  • Aktifkan Background Edit jika tersedia.
6. Stop condition
  • Dependency baru, config deploy, env/secrets, delete file, atau >3 file berubah: stop dan minta approval.
7. Gunakan /newtask saat mulai drift
  • Task panjang, error berulang, atau konteks sudah kotor.
  • Minta distilled handoff dulu.
8. Docs agent dari Git diff
  • @git-changes → documentation impact map → approval → update README/docs/CHANGELOG.
9. Kurangi context noise
  • Gunakan .clineignore untuk build output, lock noise, generated files, dan secret files.

Awareness initialization
Chief in Command requests Cline rules awareness initialization.

Task:
Read AGENTS.md, .agent/, .clinerules/, .cursor/rules/, .codex/, and .clineignore if present.

Rules:
- Do not edit files.
- Do not create files.
- Do not run destructive commands.
- Produce a concise Cline Awareness Report.

End with:
Awareness initialized. Waiting for Chief in Command's next implementation instruction.
Safe scoped implementation
Use repository AI rules.

Task:
[describe the task]

Rules:
- Start in Plan Mode.
- Inspect relevant files first.
- Do not edit yet.
- List exact files to change.
- Explain risks and rollback path.
- Mention verification command.
- Wait for approval before Act Mode.
Docs from Git changes
/docs-from-git-changes.md

Use @git-changes as the main context.

Goal:
Analyze current uncommitted changes and recommend documentation updates.

Rules:
- Do not edit yet.
- Produce documentation impact map first.
- Wait for approval before writing.
Debug root cause
@problems @terminal

Analyze this error.

Rules:
- Do not edit files yet.
- Find the root cause first.
- Show affected files.
- Propose the smallest safe fix.
- Provide verification command.
- Stop and wait for approval.
Pre-push audit
/audit-before-push.md

Audit current changes before push.

Rules:
- Do not commit.
- Do not push.
- Check git status and diff.
- Identify unrelated changes and secrets risk.
- Run relevant verification if safe.
- Suggest a commit message.
Mini persona guard
You are Sentra Engineering Guardian for Chief in Command.

Priorities:
1. Safety over speed.
2. Minimal scope.
3. No unrelated refactor.
4. No new dependency without justification.
5. Verify before claiming success.
6. Explain technical decisions in clear Bahasa Indonesia.
7. Stop and ask approval when risk increases.
