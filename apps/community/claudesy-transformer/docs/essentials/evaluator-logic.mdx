The Evaluator is a core system component that provides objective, quantifiable feedback on prompt quality using an **LLM-as-a-Judge** architecture.

## Evaluation Dimensions

The system evaluates every prompt across four primary dimensions, each weighted equally (0.25) to calculate the final `overallScore`.

### 1. Structure
Focuses on the organization and logical flow of the prompt.
- **Rubric**: 1-10 (Discernible hierarchy, clear sections, use of delimiters).
- **Weight**: 25%

### 2. Clarity
Measures language precision and the absence of ambiguity.
- **Rubric**: 1-10 (Precise language, clear intent, no contradictory instructions).
- **Weight**: 25%

### 3. Completeness
Ensures all necessary context and constraints are provided.
- **Rubric**: 1-10 (Presence of context, output specifications, and edge case handling).
- **Weight**: 25%

### 4. Specificity
Evaluates the level of concrete instruction provided.
- **Rubric**: 1-10 (Avoidance of generic requests, present of specific examples).
- **Weight**: 25%

## The Judging Mechanism

The Evaluator uses a sophisticated system prompt that forces the LLM to output a strictly structured JSON object. This ensures that the feedback is consistently parseable and fits our internal data model.

```json JSON Output Format
{
  "structure": { "score": 8, "feedback": "..." },
  "clarity": { "score": 9, "feedback": "..." },
  "completeness": { "score": 7, "feedback": "..." },
  "specificity": { "score": 8, "feedback": "..." },
  "suggestions": ["...", "..."]
}
```

## Scoring Strategy

Scores are normalized to a 0-10 scale. If an LLM response fails to parse, the system implements a **graceful degradation** strategy, defaulting to a median score (5) and providing a diagnostic feedback message.

- **Exceptional**: 9.0 - 10.0
- **Good**: 7.0 - 8.9
- **Adequate**: 5.0 - 6.9
- **Below Average**: 3.0 - 4.9
- **Poor**: 0.0 - 2.9
