Prompt Transformer is built on a high-availability infrastructure with healthcare-grade security at its core.

## Data Model (ERD)

The system uses **Prisma** to manage its PostgreSQL database in Supabase. The core models are designed for auditability and high-performance retrieval.

- **User**: Central identity with multi-provider API key storage.
- **Prompt**: Stores raw input, optimized output, and vector embeddings.
- **Evaluation**: Historical quality records linked to specific prompts.
- **UsageRecord**: Real-time tracking for subscription limits and analytics.

## Vector Infrastructure

We leverage **pgvector** to enable semantic intelligence across the platform:
- **Semantic Search**: Find similar prompts across public or private libraries.
- **Embedding Generation**: Prompts are embedded into a 1536-dimensional space (using OpenAI's `text-embedding-3-small` or equivalent).
- **Proximity Analysis**: Used to suggest similar task templates during the optimization phase.

## Security & Encryption

### API Key Protection
All user LLM provider keys are stored using **AES-256-GCM** encryption.
- **IV**: A unique Initialization Vector is generated per key.
- **Auth Tag**: Ensures data integrity and authenticity.
- **Encryption Key**: Rotated via the `ENCRYPTION_KEY` environment variable.

### Compliance & Retention
- **Soft Delete**: Records are never hard-deleted from the primary store. The `deletedAt` field ensures we maintain a full audit trail for healthcare compliance.
- **Audit Logging**: Every transformation and evaluation is logged with detailed metadata (provider, model, tokens used, latency).
- **SSL/TLS**: Mandatory encryption in transit for all data flows.
