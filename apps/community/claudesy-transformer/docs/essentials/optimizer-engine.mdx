The Optimizer Engine transforms raw ideas into high-performance "Super Prompts" by applying domain-specific strategies and structural reinforcements.

## Optimization Strategies

The engine maintains a registry of strategies tailored to specific task types. Each strategy injects unique **Emphasis Areas** and **Additional Constraints** into the transformation process.

| Task Type | Primary Focus | Key Strategy |
| :--- | :--- | :--- |
| **CODING** | Correctness & Security | Algorithmic Chain-of-Thought |
| **ANALYSIS** | Data-driven Insights | Evidence synthesis & validation |
| **MARKETING** | Conversion & Voice | Benefit-led value propositions |
| **RESEARCH** | Rigor & Coverage | Systematic evidence gathering |

## The Super-Prompt Format

The hallmark of the CTEV2 engine is the **Super-Prompt** structure. Instead of a single paragraph, it generates a modular, enterprise-grade prompt:

1.  **Role**: Defines the persona (e.g., "Expert React Architect").
2.  **Task**: A specific, actionable objective.
3.  **Context**: Necessary background information.
4.  **Approach (CoT)**: Guidance on the step-by-step reasoning.
5.  **Constraints**: Strict guardrails for the output.
6.  **Format Spec**: Exact output structure (JSON, Markdown, etc.).

## Intelligence Layer

### Template Matching
The engine uses a fuzzy-matching logic to align raw user input with established **Best Practice Templates**. This moves the prompt from a blank slate to a proven architectural pattern.

### Strategy Hints
Depending on the selected `Tone` and `Format`, the engine dynamically injects "hints" into the system prompt:
- **Tone Propagation**: Professional, Casual, Technical, etc.
- **Format Enforcement**: Detailed, Concise, Structured, Step-by-Step.
