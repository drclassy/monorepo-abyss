Security is at the heart of everything we build at Sentra Abyss.

## Healthcare Standards

Prompt Transformer follows strict guidelines to ensure healthcare data safety:

### 1. No PHI Policy
Our system is designed to never store Personal Health Information (PHI). We use automated filters to detect and redact potentially sensitive data before it reaches our database.

### 2. Audit Logging
Every action—from prompt transformation to manual edits—is logged with a unique ID and timestamp. This allows for full auditability and transparency.

### 3. Data Retention
We implement a **Soft-Delete** policy. Data is never permanently removed from our primary database; instead, it's marked as deleted to maintain a historical record for compliance audits.

## Data Encryption

- **At Rest**: All PostgreSQL databases in Supabase use AES-256 encryption at rest.
- **In Transit**: All API communication is forced over HTTPS with TLS 1.2+ protocols.
