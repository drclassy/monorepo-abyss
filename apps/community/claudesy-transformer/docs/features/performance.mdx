Prompt Transformer is built on the bleeding edge of web technology. Our architecture is optimized for low latency and high throughput.

## Next.js 15 & React 19

We use the latest features of the React ecosystem to deliver a seamless user experience:

- **React Server Components (RSC)**: Most of our dashboard is rendered on the server, reducing the JavaScript bundle size sent to the client.
- **Partial Prerendering (PPR)**: We use PPR to serve static shells immediately while streaming dynamic content as it becomes available.
- **Server Actions**: All form submissions and prompt transformations are handled via type-safe Server Actions, eliminating the need for manual API route management in most cases.

## Database Optimization

- **Prisma Accelerate**: We use Prisma's connection pooling and caching to minimize database round-trips.
- **Supabase Realtime**: Live updates for prompt history and transformation status are handled via Supabase's WebSocket-based realtime engine.

## LLM Latency Management

Processing complex prompts across multiple providers can take time. We optimize this via:

1. **Streaming Responses**: We support token streaming for OpenAI and Anthropic, allowing users to see results as they are generated.
2. **Edge Runtime**: Critical transformation logic runs on Vercel Edge Functions, placing the compute as close to the LLM providers as possible.
3. **Smart Retry Logic**: Automated exponential backoff for rate-limited requests ensures reliability without compromising speed.
